🎯 PRODUCT MISSION

Zsiistant is designed as an OpenClaw and PicoClaw competitor with these NON-NEGOTIABLES:

  • 💪 Run up to 100 agents per machine
  • 🔒 Strong isolation with explicit communication rules
  • ⚡ Stay fast, lean, and conservative with resources
  • 🌐 Support many LLM backends (Ollama, Ollama Cloud, Z.AI first!)
  • 🔌 Import or adapt OpenClaw-style skills
  • 🏪 Grow into an internal skill store
  • 🎨 Simpler operator experience with PicoClaw polish

🏆 PICOWCLAW PARITY REQUIREMENTS

The plan now explicitly assumes that Zsiistant must eventually do everything PicoClaw does, while preserving the same spirit of small-factor performance:

  • 📦 Self-contained, low-overhead operation
  • ⚡ Fast startup and low idle footprint
  • 🔌 Multi-channel connectivity
  • ⚙️ Strong settings and model management
  • 🌐 Clean Web UI
  • 🚀 Portable deployment story

💰 TOKEN DISCIPLINE

Zsiistant should be deliberately conservative with token generation and model usage:

TOKEN OPTIMIZATION: ACTIVE
  • 🎯 Prefer smaller models for routing, classification, filtering
  • 🏆 Reserve expensive models for high-value reasoning only
  • 🔄 Support fallback chains and hard caps
  • 📏 Minimize prompt bloat
  • 👀 Make model/tool routing visible and controllable
  • ⚜️ Keep autonomous runs efficient and bounded
